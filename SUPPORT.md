# Support

## Start Here

Before opening an issue, review:

- [README.md](README.md)
- [release-notes.md](release-notes.md)
- [CHANGELOG.md](CHANGELOG.md)
- `az-vm -h`
- `az-vm help <command>`

## Best Support Paths

- Use GitHub issues for clear bugs, documentation gaps, and focused feature requests that can be discussed publicly.
- Use the maintainer contact path for licensing, sponsorship, partnership, or other private coordination topics.
- Use the private reporting path in [SECURITY.md](SECURITY.md) for vulnerabilities or sensitive safety issues.

## What To Include

For normal support or bug reports, include:

- the command you ran
- expected behavior
- actual behavior
- relevant help/docs section you followed
- a sanitized `az-vm show` excerpt or lifecycle snapshot when the issue is state-related
- sanitized logs or error output
- platform details such as Windows or Linux guest target

## What Not To Post Publicly

Do not post secrets, passwords, subscription identifiers, private hostnames, or other sensitive infrastructure details in public issues.

## Response Model

This repository is maintained with a contact-first, maintainer-curated workflow. Support and review are best-effort, and priority may depend on severity, clarity, and maintainer availability.
